Оформление

Когда идёт речь о вебсайте, все сразу представляют прямоугольники наложенные друг на друга разных цветов, и это абсолютно так. И поэтому разработчику и дизайнеру нужно исхитряться, чтобы придумать что-нибудь такое, что зацепит внимание посетителя сайта, что завлечёт его остаться и воспринять информацию. Тут на помощь приходят разные методы для оформления простых фигур и текста.
Сложные Фигуры

Так как все элементы в HTML изначально прямоугольники, у разработчиков задача оформить их так, как хочет дизайнер.

Среди методов изменения формы элементов можно выделить наиболее популярные.

Например:

1) Скругление краёв параметром border-radius

Для данного примера, я назначил параметру border-radius у квадрата значение 50%, которое скругло углы квадрата с радиусом равным пловине стороны

2) Наложение элементов

Здесь я наложил на круг из прошлого примера второй, чей цвет идентичен фоновому

3) Параметр clip-path

В Данном примере я использую параметр clip-path с сочетанием метода polygon() который позволяет задать двухмерные координаты точек. Для данной фигуры значение clip-path выглядит так:

polygon(0 0, 100% 0, 100% 40%, 70% 40%, 50% 25%, 25% 50%,
        50% 75%, 70% 60%, 100% 60%, 100% 100%, 0 100%)

Но всё же этот параметр чаще используется для разделения текста.


У этих методов есть и свои минусы. Скругление границы элемента также скругляет и обводку. Наложение плодит необязательные элементы, а вырезать из объекта произвольную фигуру вовсе не рекомендуется. 2 и 3 методы загружают оперативную память браузера, а это ведёт ко многим другим плохим последствиям. Тут надо выбирать между производительностью и дизайном или вовсе заниматься скучнейшей оптимизацией.

 Анимации

Анимации это неотъемлемая часть pure CSS. На них строится все изменения элементов, которые на самом деле невероятно просты.

Анимации это ключ @keyframes [имя анимации] {параметры}. С его помощь определяется то, как изменяется какое-либо значение параметра. После создания анимации её можно применить к любому элементу с помощью параметра animation. Вместе с именем анимации передаются значения длительности анимации, задержка, сколько раз повторяется анимация, в какую стороны идёт воспроизведение анимации и временная функция.

Этот кусок кода соответствует анимации примера

1    @keyframes demo-animation {
2      0%,100% {
3        background-color: red; top: 5%; left: 40%;
4      }
5      25% {
6        background-color: cyan; top: 40%; left: 75%;
7      }
8      50% {
9        background-color: blue; top: 75%; left: 40%;
10     }
11     75% {
12       background-color: pink; top: 40%; left: 5%;
13     }
14   }
Хитрости

Так как нет кода JavaScript расчитывать на сложную логику кода не приходится. Здесь применяются хитрости. Здесь мне даже отдельно не прийдётся делать пример т.к. навигация по сайту это одна сплошная хитрость. На самом деле боковая панель "видна" всегда, но находится она на 4 экрана правее :D

Для того чтобы логика открытия и закрытия боковой панели работала, была использована древняя хитрость. А именно обычная невидимая кнопка-флажок. Когда пользователь переключает её состояние CSS использует значение псевдокласса :checked как переменную. Если кнопка нажата, то применить значения параметров при нажатой кнопке, когда нет - применить стилизацию этого элемента без изменений. На этом вся хитрость и заканчивается, и начинается обычное написание стилей.
